# AGENTS.md

## 핵심 원칙

- 요청 범위에 필요한 변경만 한다.
- 기존 구현을 먼저 읽고, 중복 유틸이나 컴포넌트를 만들지 않는다.

## Package Manager: pnpm ONLY

- `npm`이나 `yarn`을 사용하지 않는다. lockfile 충돌을 막기 위해 `pnpm`만 사용한다.
- 의존성 설치는 `pnpm install`을 사용한다.
- 개발 서버는 `pnpm dev`로 실행한다.

## 코드 품질

- 코드 흐름은 단순한 분기로 읽기 쉽게 만든다.
- 의도가 드러나는 이름을 사용하고 `data`, `temp`, `flag`처럼 모호한 이름을 피한다.
- 한 함수나 컴포넌트가 여러 책임을 갖기 시작하면 분리 여부를 검토한다.

## React 규칙

- Hooks는 항상 컴포넌트 최상위에서 호출한다.
- 조건부 렌더링은 모든 hook 호출 이후 early return으로 처리한다.
- 렌더링 중 계산할 수 있는 값은 `useState`와 `useEffect`로 동기화하지 않는다.
- 컴포넌트는 하나의 UI 책임에 집중시킨다.
- 컴포넌트나 hook 추출은 실제 중복이나 복잡도를 줄일 때만 한다.

## TypeScript

- 린트 규칙을 비활성화하거나 타입 검사를 우회해야 한다면, 먼저 설계를 다시 검토한다.
- 타입 오류를 숨기지 말고 타입 좁히기, 명시적 타입, 데이터 구조 개선으로 해결한다.

## 안전 규칙

- 로딩, 빈 상태, 에러 상태는 빠뜨리지 않고 명시적으로 렌더링한다.
- 콘솔 출력은 디버깅 로그로 남기지 않는다. 필요한 경고/오류 출력은 의도를 드러내는 에러 처리 흐름 안에서만 사용한다.

## 검증

- 커밋 전 staged 파일은 `lint-staged`로 ESLint와 Prettier 검사를 통과해야 한다.
- push 전 `pnpm typecheck`를 통과해야 한다.
- 코드나 설정을 변경한 뒤에는 `pnpm lint`, `pnpm format:check`, `pnpm typecheck` 중 관련 검증을 실행한다.
- `--no-verify`로 git hook을 우회하지 않는다.
