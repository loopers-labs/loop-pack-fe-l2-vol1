## 배경

**집들이** 구경하고, 마음에 드는 공간을 **스크랩**하고, 그 공간 속 아이템을 바로 **구매**하는 커머스.

"예쁜 방 사진"에서 시작해 "이 조명 어디서 샀어?"까지 자연스럽게 연결되는 서비스.

우린 이 흐름을 Phase별로 직접 만들어갈 거예요.

## 기술 스택

| Phase | 스택 | 이유 |
| --- | --- | --- |
| **Phase 0·Phase 1** (사전∼R3) | Vite + React | 독립 카타·원칙 학습에 적합. 프레임워크 무관한 React 기본기에 집중 |
| **Phase 2·Phase 3** (R4∼R10) | **Next.js App Router** | "예쁜 방 사진"은 검색 노출(SEO)이 필수 → SSR/SSG. 이미지가 핵심 콘텐츠 → `next/image` 자동 최적화가 실무 이익 |

> **왜 R4에서 전환하는가**: Phase 1의 "독립 카타 — 프레임워크 무관 원칙" 철학을 지키면서, 프로젝트 착수 시점에 **"왜 프레임워크가 필요해지는가"를 체감**하고 도입하도록 설계. 멘티는 React/Next의 경계를 명확히 인식한 상태로 실무에 진입하게 됩니다.

## 공개 시점

이 문서는 한 번에 전부 공개되지 않습니다. 프로젝트 진행에 맞춰 단계적으로 공개됩니다.

| 시점 | 공개 범위 | 이유 |
| --- | --- | --- |
| **Phase 0~1 (사전~R3)** | 공개하지 않음 | 독립 카타에 집중 |
| **R4 시작 시** | 서비스 컨셉 + 콘텐츠/상품 영역 + 기본 데이터 모델 | 프로젝트 착수에 필요한 최소 범위 |
| **R5 시작 시** | 커머스 영역 (장바구니/주문) + 사용자 영역 | 상태 관리 주제와 맞물려 복잡한 상태 흐름 필요 |
| **R6 이후** | 추가 공개 없음 | 테스트·설계·성능·배포 단계 |

> **왜 단계적으로 공개하는가?**
요구사항이 나중에 추가되는 경험 자체가 실무와 동일하며, R8(시스템 설계)의 자연스러운 동기가 됩니다.
>
