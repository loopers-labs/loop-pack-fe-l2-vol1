# PR 디스크립션 작성

TRIGGER — PR 디스크립션 작성, PR 본문 작성 요청 시 이 스킬을 먼저 로드.

## 템플릿

`.github/pull_request_template.md`를 반드시 따른다.

## 자동 수집 (사용자 입력 불필요)

아래 항목은 git 정보에서 자동으로 추출한다:

- **주차**: 브랜치명에서 추출 (예: `feat/week-01-...` → `Week 01`)
- **무엇을 / 왜**: `git log main..HEAD`와 `git diff main...HEAD --stat` 기반으로 초안 작성, 사용자 확인 후 확정
- **AI 생성 범위**: 본문 끝에 자동 추가

## 사용자 입력 필수

아래 항목은 사용자에게 반드시 질문하여 받는다:

1. **학습 주제**: 이번 과제를 통해 학습한 주제
2. **배운 것 / 새로 적용한 것**: 새로 배우거나 적용해 본 내용
3. **고민한 점 / 막혔던 부분**: 선택을 두고 고민한 것, 어려웠던 것, 미해결 사항
4. **피드백 받고 싶은 부분**: 리뷰어가 집중해서 봐줬으면 하는 부분

## 작성 절차

1. `git log`, `git diff --stat`으로 변경 사항 파악
2. 사용자에게 필수 입력 4가지 질문
3. 템플릿에 맞춰 디스크립션 조합
4. 본문 끝에 AI 생성 범위 한 줄 추가 (예: "AI: Claude Code로 초안 작성 후 수동 편집")
